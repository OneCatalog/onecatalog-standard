# История изменений — OneCatalog Integration Standard

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
нумерация версий — по [семантическому версионированию](https://semver.org/lang/ru/).

Версионируется сам **стандарт интеграции**. Платформенные порты указывают, какой
версии стандарта они соответствуют.

## [Не выпущено] — бэклог

### Добавлено (на `dev`) — будет релизом 1.1.0
- **§13. Синхронизация цен и остатков (B2B-фид)** — новый платформо-независимый
  сценарий поверх импорта каталога (внедрён в эталоне — WooCommerce 1.8.0):
  отдельный B2B API и авторизация (`url_key` в пути + `private_key` в query);
  формат `known`/`unknown` + справочники `regions`/`warehouses`; маппинг (known по
  `public_id`, unknown по коду поставщика); стратегии цены × приоритет регионов;
  суммирование остатков по складам, дробные остатки; **change-detection (scan-and-diff)**
  как обязательный инвариант нагрузки; ненайденные (импорт через очередь / skip /
  создать+код); расписание; точки расширения для раскладки цен по регионам и остатков
  по складам в сайтовые поля; чек-лист переноса.
- §13.5 расширен: для `unknown` добавлен режим **«отстойник» (staging) — ручной отбор**
  (хранилище + страница: импортировать / игнорировать, статусы `new|imported|ignored`)
  как «шлюз» против засорения каталога; рядом с режимами skip и авто-создание.
- Интро: стандарт явно покрывает **два сценария** — импорт каталога (§1–§12) и
  синхронизацию цен/остатков (§13).

### Бэклог
- Раздел про мультиязычность (общий пробел: API отдаёт один язык за запрос; модель
  «один объект + переводы» против «объект на язык» — отдельная фаза для всех портов).
- Опциональный общий PHP-core (Composer-пакет) для PHP-семейства портов
  (WooCommerce / PrestaShop / Bitrix / OpenCart / Magento): API-клиент, конвертер
  единиц, контент-ключ/дедуп, нормализация payload.

## [1.0.1] — 2026-06-16

## [1.0.0] — 2026-06-16

### Добавлено
- Первая публичная версия стандарта (`INTEGRATION-STANDARD.md`): внешние контракты
  Wiki API (§2), доменная модель и маппинг (§3), слои архитектуры (§4), обязательные
  инварианты (§5), фоновая очередь (§6), настройки (§7), точки расширения (§8), i18n
  (§9), жизненный цикл (§10), чек-лист переноса (§11), карта переноса Bitrix/PrestaShop (§12).
- Планы портирования (`integration-reviews/`) по 18 платформам с ответами (факты
  живого API + рекомендации эталона).
